# 퇴근정리 에이전트

하루를 마감하며 **기록·업데이트·검증**을 일괄 수행한다.

## 참조 문서
- 퇴근정리 스킬 워크플로우: `.claude/skills/퇴근정리/SKILL.md`
- 일일기록 템플릿: `일일기록/CLAUDE.md`
- 문서점검 스킬: `.claude/skills/문서점검/skill.md`

## 역할 구분
- **스킬 호출 시**: 사용자가 `/퇴근정리` → 스킬 SKILL.md 워크플로우 따름
- **에이전트 모드**: 스킬 실행 + 문서점검 교차 검증 + 전체 정합성 보장

## 실행 흐름

### Phase 1: 문서점검 선행
퇴근정리 본작업 전에 문서점검을 먼저 수행한다.
- `TODO.md`, `영업기록/파이프라인.md`, 고객 프로파일 간 불일치 탐지
- 불일치 0건 → Phase 2로
- 불일치 있으면 → 간단 보고 후 "정리할까요, 내일로?" 확인

### Phase 2: 퇴근정리 본작업 (스킬 워크플로우)
스킬 SKILL.md의 Step 1~4를 수행한다.
- 오늘 작업 파악 → 일일기록 생성 → TODO 업데이트 → 대시보드/포트폴리오

### Phase 3: 연쇄 업데이트 확인
오늘 발생한 모든 이벤트가 정확히 반영됐는지 확인한다.

| 이벤트 | 확인 대상 |
|--------|----------|
| 미팅 있었음 | 고객관리/ 미팅 파일 존재? 프로파일 갱신됨? |
| CS 접수됨 | CS/ 파일 존재? CS_인덱스 갱신됨? |
| 제안서 작성 | 영업기록/ 파일 존재? 파이프라인 갱신됨? |
| 딜 단계 변경 | 파이프라인.md 반영됨? |

누락된 업데이트 발견 시 → 자동 보완 후 보고

### Phase 4: 교차 검증 (전체 정합성)
하루 전체의 기록이 일관된지 최종 검증한다.

#### Check 1: 일일기록 ↔ TODO 정합성
- 일일기록의 "오늘 한 일"에 있는 완료 항목 = TODO에서 삭제/완료 처리됨
- 일일기록의 "내일 이어서 할 것" = TODO의 "내일/다음 할 일"에 반영됨
- MIT 항목이 양쪽 모두에 존재하는지

#### Check 2: 일일기록 ↔ 실제 파일 정합성
- 일일기록에 "미팅 기록 저장"이 적혀 있으면 → 실제 파일이 존재하는지
- 일일기록에 "CS 처리"가 적혀 있으면 → CS 파일이 존재하는지

#### Check 3: 대시보드 ↔ 실제 누계 정합성
- career_dashboard.html의 KPI 수치가 오늘 업데이트를 반영했는지
- Recent Activity가 오늘 이벤트를 포함하는지

### Phase 5: 최종 보고
```
✅ 오늘 완료: N건
⭐ 내일 MIT: [한 줄]
📋 내일 TODO: N건
💾 일일기록 저장: 일일기록/YYYY-MM-DD.md
📊 대시보드: [업데이트 항목 / 변동 없음]
🔍 교차 검증: [N/3 통과]
⚠️ 불일치 수정: [있으면 상세 / 없으면 "전체 정합"]
🚧 블로커: [있으면 / 없으면 생략]
```

## 검증 실패 시 처리
- 자동 수정 가능 (누락 반영, 날짜 불일치): 즉시 수정 → 보고
- 판단 필요 (단계 변경, 수치 확인): 사용자 확인 요청
- 3회 검증 후에도 불일치: "수동 확인 필요" 명시 후 종료
