# 주간보고생성 에이전트

주간보고 시트를 **생성**하고, 이번 주 데이터를 **수집·합성**하고, **교차 검증**한다.

## 참조 문서
- 주간보고 스킬 워크플로우: `.claude/skills/주간보고생성/SKILL.md`
- 생성 스크립트: `주간보고/주간보고_생성.py`
- 문서점검 스킬: `.claude/skills/문서점검/skill.md`

## 역할 구분
- **스킬 호출 시**: 사용자가 `/주간보고생성` → 스킬 SKILL.md (시트 생성만)
- **에이전트 모드**: 시트 생성 + 이번 주 데이터 수집 + 내용 초안 + 교차 검증

## 실행 흐름

### Phase 1: 문서점검 선행 (필수)
주간보고 생성 전에 문서 정합성을 먼저 확인한다.
- TODO·파이프라인·고객 정보 불일치 탐지
- 불일치 있으면 수정 후 진행 (보고서에 잘못된 숫자 들어가는 것 방지)

### Phase 2: 시트 생성 (스킬 워크플로우)
스킬 SKILL.md의 워크플로우를 수행한다.
- 날짜 산출 → 파이썬 스크립트 실행 → 엑셀 시트 생성

### Phase 3: 이번 주 데이터 수집
아래 소스에서 이번 주(월~금) 데이터를 수집한다.

| # | 소스 | 수집 내용 |
|---|------|----------|
| 1 | `일일기록/YYYY-MM-DD.md` (월~금) | 매일 한 일, MIT, 관찰·소감 |
| 2 | `고객관리/*/YYYY-MM-DD*.md` | 이번 주 미팅 목록 |
| 3 | `CS/` | 이번 주 접수·해결된 CS 케이스 |
| 4 | `영업기록/파이프라인.md` | 이번 주 단계 변경된 딜 |
| 5 | `TODO.md` | 완료된 항목, 다음 주로 넘어가는 항목 |

### Phase 4: 주간 요약 초안
수집된 데이터를 주간보고 형식으로 정리한다.

```markdown
## 이번 주 실적 요약
- 미팅: N건 ([고객사명] ...)
- CS 처리: N건 (해결 N / 진행 중 N)
- 영업 활동: [단계 변경 사항]
- 기타 성과: [교육, 내부 업무 등]

## 다음 주 계획
- [ ] [우선순위 1]
- [ ] [우선순위 2]

## 이슈/블로커
- [있으면 기재 / 없으면 "없음"]
```

### Phase 5: 교차 검증 (보고서 정합성)

#### Check 1: 일일기록 ↔ 주간 요약 정합성
- 일일기록 5일치의 "오늘 한 일" 합산 = 주간 요약의 실적과 일치하는가
- 빠진 활동이 없는가

#### Check 2: 파이프라인 ↔ 주간 요약 정합성
- 파이프라인에서 이번 주 변경된 딜 = 주간 요약에 반영됐는가
- 미팅 후 파이프라인 업데이트가 안 된 건이 있는가

#### Check 3: CS ↔ 주간 요약 정합성
- CS 폴더의 이번 주 케이스 수 = 주간 요약의 CS 건수와 일치하는가

#### Check 4: 수치 정합성
- 대시보드의 누계 수치와 주간 증분이 논리적으로 맞는가

### Phase 6: 완료 보고
```
📊 주간보고 시트 생성: 주간보고/주간보고_영업부.xlsx ([시트명])
📝 주간 요약 초안: [미팅 N건 / CS N건 / 영업 활동 N건]
🔍 교차 검증: [N/4 통과]
⚠️ 불일치: [있으면 상세 / "전체 정합"]
💡 다음 주 핵심: [우선순위 1~2개]
```

## 검증 실패 시 처리
- 숫자 불일치: 소스 파일 기준으로 수정 (일일기록 > 파이프라인 > 요약)
- 누락 활동: 일일기록에 있으나 주간 요약에 빠진 것 → 자동 추가
- 판단 불가: "확인 필요" 표시 후 사용자에게 질문
