# 에이전트 라우터

모든 사용자 메시지에서 아래 패턴을 감지하면 즉시 해당 에이전트/스킬을 활성화한다.

## 라우팅 규칙

### 미팅기록 에이전트/스킬 트리거
다음 중 하나라도 감지되면:
- **기본**: `미팅기록` **에이전트** 활성화 (기록 + 연쇄 업데이트 + 교차 검증)
- **간단 기록만 필요** (사용자가 "간단히", "기록만"): `미팅기록` 스킬 실행

트리거 키워드:
- "미팅", "만났어", "방문", "미팅했어", "고객 만나서"
- "[고객사명] 갔다왔어", "[고객사명] 미팅"
- "오늘 [회사명]에서", "미팅 내용 정리"

### CS케이스 스킬/에이전트 트리거
다음 중 하나라도 감지되면:
- **단순 기록** (채팅 복붙, 이슈 1건) → `/CS케이스` 스킬 실행
- **복합 분석** (해결책 필요, 패턴 분석, 여러 이슈) → `CS케이스` 에이전트 활성화

트리거 키워드:
- "CS", "문의", "오류", "안 된다", "안 돼", "고장"
- "연결이 안", "설정이 안", "작동이 안", "기술지원"
- "클레임", "VOC", "불만", "에러"
- "이슈", "저장 안 됨", "문의 들어왔어", "고객이 연락왔어"

트리거 키워드:

## 협업형 대화 모드 감지
아래 패턴이면 해당 스킬/에이전트를 **협업형 모드(Step 0)로 시작**하도록 라우팅:
- "제안서 써줘", "견적 보내야 해", "영업 전략" → 영업제안 에이전트 (Step 0: 전략 대화)
- "제안서 PPT 만들어", "제안 슬라이드" → PPTX생성 스킬 (Step 0: 구조 합의)
- "첫 메일", "제안 메일", "견적서 메일" → 메세지 스킬 (Step 0: 전략 질문)
- "어떻게 접근할까", "전략 짜줘", "방향 잡아줘" → 일반 대화 + 협업형 5단계 프로세스

**즉시 실행 유지 (협업형 아님):**
- CS 접수, 미팅 기록, 정산, 일정 → 에이전트/스킬 즉시 실행 (팩트 기반)
- 사용자가 "바로", "빨리", "그냥 써줘" 명시 → 협업형 건너뛰기

**에이전트 vs 스킬 라우팅 원칙:**
- **기본은 에이전트** (해당 에이전트가 있는 경우) — 연쇄 업데이트 + 교차 검증까지 수행
- **스킬만 실행**: 사용자가 "간단히", "빠르게", "기록만" 등 경량 처리 명시 시
- 에이전트가 없는 스킬 (메세지, 일정, 정산 등) → 기존대로 스킬 실행
