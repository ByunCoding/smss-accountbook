# 스크립트분석

## 목적
유튜브 전문가 영상 스크립트를 분석하여 `스크립트/` 라이브러리에 구조화 저장.
업무에 바로 쓸 수 있는 형태로 핵심만 추출한다.

## 트리거 조건
아래 중 하나 이상 + 파일 경로가 함께 있을 때:
- 유튜브 스크립트/자막 파일(.md, .txt)이 제공됨
- "이거 정리해", "스크립트 폴더에 넣어줘", "분석해줘"
- "영상 정리", "유튜브 정리", "이거 추가해"
- 파일 내용에 YouTube URL + Transcript 패턴이 있을 때

## 실행 순서

### Step 1: 파일 읽기 + 메타 정보 추출
파일을 읽고 아래 정보를 파악한다:
- **제목**: 파일 첫 줄 또는 파일명
- **출처 URL**: YouTube URL (있으면)
- **화자/전문가**: 누가 말하고 있는지 (영상 제목·내용에서 추정)
- **주제 분류**: 어떤 스크립트 폴더에 들어갈지 판단

### Step 2: 폴더 배치 결정
`스크립트/CLAUDE.md`의 **폴더 → 질문 키워드 매핑 테이블**을 읽고:
1. 기존 폴더에 맞는 주제인지 확인
2. 맞으면 → 해당 폴더에 배치
3. 안 맞으면 → 새 폴더 생성 제안 (사용자 확인 후 생성)

배치 판단 시 사용자에게 간단히 보고:
```
📂 배치: [폴더명] — [이유 한 줄]
```

### Step 3: 원본 파일 복사
원본 파일을 해당 폴더로 복사한다.
- 파일명은 원본 그대로 유지

### Step 4: 핵심 분석 + 📋_분석요약.md 업데이트
해당 폴더의 `📋_분석요약.md`를 읽고, 아래 구조로 새 섹션을 추가한다:

```markdown
## [이모지] [핵심 주제 한 줄] ([화자/출처])

> 핵심: **[한 문장 요약]**

### [내용 구조에 맞게 자유 구성]
- 표, 리스트, 단계별 등 내용에 가장 맞는 형태로
- 노이즈(인트로, 광고, 중복, 잡담) 제거 → 실전 팁만 추출
- 원본 대비 30~40% 분량으로 압축

### 와트 적용
| 상황 | 적용 방법 |
|------|---------|
| [현재 업무에 맞는 상황] | [구체적 적용 방법] |
```

**분석 시 참조할 컨텍스트**:
- `CLAUDE.md` → 나의 포지션, 역할, 업무 구조
- `참조자료/고객.md` → 현재 고객사 상황 (와트 적용 작성 시)
- 사용자의 커리어 전략 (Phase 1~3) → 지금 단계에 맞는 적용 포인트 강조

### Step 5: 📂 원본 파일 목록 업데이트
`📋_분석요약.md` 하단의 `## 📂 원본 파일` 섹션에 새 파일 추가:
```
- `파일명.md` — [핵심 키워드 나열]
```

### Step 6: 스크립트/CLAUDE.md 업데이트
3가지를 업데이트한다:
1. **폴더 → 질문 키워드 매핑**: 새 키워드가 있으면 추가
2. **현재 파일 현황 테이블**: 파일 수 +1, 주요 출처에 화자 추가
3. **새 폴더 생성 시**: 매핑 테이블에 새 행 추가

### Step 7: 검증
- [ ] 원본 파일이 올바른 폴더에 복사됐는지 확인
- [ ] 📋_분석요약.md에 새 섹션이 추가됐는지 확인
- [ ] 📂 원본 파일 목록에 등록됐는지 확인
- [ ] 스크립트/CLAUDE.md 파일 수·키워드가 갱신됐는지 확인

### Step 8: 완료 보고
```
📥 분석 완료: [영상 제목]
📂 배치: 스크립트/[폴더명]/
📝 요약: [핵심 한 줄]
🏷️ 키워드: [추가된 키워드들]
⭐ 와트 적용: [가장 중요한 적용 포인트 1개]
```

## 복수 파일 처리
여러 파일이 동시에 들어오면:
1. 각 파일별로 Step 1~2 (폴더 배치)를 먼저 결정
2. 같은 폴더에 들어가는 파일은 묶어서 처리
3. 각 파일별 완료 보고 후, 전체 요약 1줄 추가

## 품질 기준
- **노이즈 제거**: 인트로, 아웃트로, 광고, 구독 유도, 중복 설명 → 삭제
- **구어체 → 핵심 정리**: 말투·장황한 예시 → 원칙·프레임워크로 압축
- **와트 맞춤**: 스마트글라스 기술영업 + 스타트업 상황에 맞는 적용 포인트 필수
- **기존 요약과 톤 맞추기**: 해당 폴더의 기존 📋_분석요약.md 스타일을 따름
