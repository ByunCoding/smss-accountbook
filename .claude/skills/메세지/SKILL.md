# 메세지

> 김우곤 부장님 글쓰기법 기반. 격식 있되 딱딱하지 않은 톤.
> 부장님 실제 메일이 추가될 때마다 템플릿·표현 사전을 누적 업데이트한다.

## 실행 순서

### Step 0: 협업형 판단 (전략적 메일일 때만)
아래 조건이면 바로 쓰지 않고 전략 질문을 1~2개 던진다:
- **첫 제안 메일** (고객과 처음 연락)
- **견적서·제안서 발송 메일** (비용·결정 관련)
- **거절·난처한 상황 대응** (클레임, 가격 협상)

**일상 메일·단순 공유·내부 메신저** → Step 0 생략, 바로 Step 1로

### Step 1: 기본 정보 파악
사용자 입력에서 아래를 파악 (모르면 물어본다):
- **받는 사람**: 이름 + 직책
- **채널**: 메신저 / 카톡 / 이메일 / 문자 (기본값: 메신저)
- **용건**: 전달 내용
- **요청사항**: 받는 사람에게 원하는 액션
- **첨부 내용**: 그대로 보낼 본문이 있는지

### Step 2: 글쓰기법에 맞춰 초안 작성
아래 **김우곤 부장님 글쓰기법** 섹션의 원칙 → 대상별 톤 → 유형별 템플릿 → 표현 사전 순서로 적용하여 초안을 작성한다.

### Step 3: 첨부 내용 처리
사용자가 "이 내용 그대로 보낼 거야"라고 한 본문이 있으면:
- 인사 + 용건/요청만 앞에 붙이고
- 원본 내용은 **수정 없이** 그대로 아래에 배치

### Step 4: 초안 출력 + 즉시 저장 (MUST)
- 대화창에 초안을 출력하는 **동시에** `메일.md`에 무조건 덮어쓰기 저장한다. 사용자 확정 대기 금지.
- 저장 경로: `C:\Users\user\Documents\GitHub\WATT\메일.md` (매번 덮어쓰기)
- 확인 필요한 부분은 `[  ]`로 표시 (예: `[시간 확인 필요]`)

### Step 5: 수정 반복
사용자가 "이렇게 바꿔" 하면 즉시 반영해서 재출력 + `메일.md` 재덮어쓰기.
패턴이 반복되면 메모리에 저장하여 다음부터 자동 반영.

## 김우곤 부장님 글쓰기법

### 5대 원칙

| # | 원칙 | 설명 |
|---|------|------|
| 1 | **접촉 경위 먼저** | 왜 연락하는지 한 줄로 밝힌다 ("미팅에서 말씀하셨던", "요청하신") |
| 2 | **용건 한 줄** | 인사 직후 바로 본론. 읽는 사람이 첫 3줄만 봐도 파악 가능 |
| 3 | **번호 리스트 구조화** | 본문은 항목별 소제목+내용. 훑어보기 쉽게 |
| 4 | **고객 배려 한 줄** | 왜 이렇게 구성했는지, 고객 입장에서 한 줄 설명 |
| 5 | **마무리 3단** | 첨부 확인 요청 → 추가 문의 안내 → "감사합니다." |

### 이메일 기본 뼈대

```
안녕하세요, {이름} {직책}님. (주)와트 변상민 대리입니다.

{접촉 경위 + 용건 한 줄}

주요 내용은 아래와 같습니다.

1. {항목 소제목}
   {내용}

2. {항목 소제목}
   {내용}

{마무리 — 첨부 확인 / 추가 문의 안내}

감사합니다.
```

### 대상별 톤

| 대상 | 톤 | 인사 형식 |
|------|-----|----------|
| **사내 윗분** (이사님, 부장님, 차장님) | 정중 + 간결 | "안녕하세요 차장님, 영업부 변상민입니다." |
| **사내 동료** (같은 직급) | 편한 존댓말 | "안녕하세요, 변상민입니다." |
| **고객사 일반** | 격식 존댓말 | "안녕하세요, {이름} {직책}님. (주)와트 변상민 대리입니다." |
| **공기업 고위직** | 최고 격식 | "{이름} {직책}님, 안녕하십니까. (주)와트 변상민 대리입니다." |

### 채널별 기준

| 채널 | 길이 | 특징 |
|------|------|------|
| **이메일** | 구조화 | 제목 필수, 번호 리스트, 섹션 구분 |
| **메신저/카톡** | 짧게 | 핵심만, 스크롤 최소화 |
| **문자** | 초간결 | 2~3줄 이내 |

#### 채널별 표현 차이

| 상황 | 이메일 | 메신저/카톡 |
|------|--------|-----------|
| 용건 시작 | "~건과 관련하여 안내드립니다" | "~건으로 연락드립니다" |
| 회신 | "회신드립니다" / "답변드립니다" | "확인해서 말씀드립니다" |
| 요청 | "~부탁드리겠습니다" | "~부탁드립니다" |
| 끝인사 | "감사합니다." | "감사합니다!" / "확인 부탁드립니다" |

> "연락드립니다"는 전화/카톡 톤. 이메일에서는 "안내드립니다/회신드립니다" 사용.

### 이메일 유형별 서식

| 유형 | Bold | 적용 위치 |
|------|------|----------|
| 견적서·제안서 발송 | O | 소제목, 핵심 수치(금액·기간·수량), 항목명 |
| 일반 안내·회신 | X | 텍스트만. 간결함이 포인트 |
| 개발 일정·요청사항 안내 | O | 항목 번호 소제목, 일정(날짜), 변경 사항 |
| 정기 보고·실적 업데이트 | O | 비교 수치(기존→업데이트), 항목별 증감, 섹션 소제목 |

> 의사결정자가 훑어봐야 하는 내용(금액, 일정, 스펙)은 Bold. 단순 안내는 Plain.

### 이메일 제목 형식

`[주식회사 와트] {고객사} {제품/건명} {용건}의 건`

## 유형별 템플릿

> 부장님 실제 메일이 추가될 때마다 여기에 새 템플릿을 누적한다.

### 1-A. 견적서 발송 — 옵션형 (비교 구성: 렌탈/구매, 모델 비교 등)

> 사용 시점: 고객이 **의사결정을 위해 복수 안을 비교**해야 할 때 (운영 방식 선택, 도입 규모 비교)

```
안녕하세요 {고객사} {이름} {직책}님. {접촉 경위} (주)와트 변상민 대리입니다.

{용건 한 줄 — 요청하신 ~ 보내드립니다.}

{옵션 구성 이유 — 귀사의 운영 방식에 맞춰 선택하실 수 있도록 ~}
주요 내용은 아래와 같습니다.

1. 견적 개요 (VAT 별도)
   {옵션A} : {금액} ({포함 내역 요약})
   {옵션B} : {금액} ({조건 요약})

2. 공통 포함 내역
   하드웨어 : {모델명} ({구성품})
   소프트웨어 : {솔루션명} ({기능 요약})
   통신 : {요금제}

상세한 품목별 단가는 첨부해 드린 PDF 견적서를 확인해 주시기 바랍니다.
검토 후 추가 문의 사항이나 협의가 필요한 부분이 있으시면 언제든 편하게 연락 주십시오.
긍정적인 검토 부탁드립니다.

감사합니다.
```

### 1-B. 견적서 발송 — 대형 프로젝트 (단일 구성)

> 사용 시점: **사업 단위의 대형 프로젝트 견적** — 시스템 구축·통합 솔루션처럼 구성이 단일안이고, 의사결정자가 사업의 골자를 먼저 이해해야 할 때
> 레퍼런스: 2026-04-17 한일시멘트 통합안전관리 플랫폼 CCTV 시스템 구축 견적 (김우곤 부장)

```
{고객사} {이름} {직책}님 안녕하세요. (주)와트의 변상민 대리입니다.

'{사업명}' 사업과 관련하여, 요청하신 견적서를 첨부와 같이 송부드립니다.

이번 프로젝트는 {핵심 가치·목적 한 줄}을(를) 골자로 하고 있습니다. 주요 내용은 다음과 같습니다.

1. 견적 개요
   - 견적 금액 : ₩ {금액} (VAT 별도)
   - 납품 기간 : 계약 후 {기간} 이내

2. 기타 안내 사항
   - 본 견적의 유효기간은 발행일로부터 {일수}일입니다.
   - {리스크 고지 — 예: 현장 상황에 따라 전기 배선 공사나 캐비닛 설치 등 추가 인프라 비용이 발생할 수 있습니다.}

첨부된 견적서를 검토해 주시고, 추가 설명이 필요하시거나 조정이 필요한 부분이 있다면 언제든 편하게 말씀해 주십시오.
감사합니다.
```

**1-A vs 1-B 선택 기준**

| 구분 | 1-A (옵션형) | 1-B (대형 프로젝트) |
|------|-------------|------------------|
| 견적 금액 규모 | 수백만~수천만 원대 표준 도입 | 억 단위 대형 구축 사업 |
| 구성 | 복수 옵션 비교 | 단일안 |
| 본문 핵심 | 옵션별 금액 + 공통 포함 내역 | 사업 골자 + 견적 개요 + 리스크 고지 |
| 마무리 강조 | "긍정적인 검토 부탁드립니다" | "조정이 필요한 부분이 있다면 편하게 말씀" (협의 여지 오픈) |
| 리스크 고지 | 보통 생략 | **필수** — 현장 변수로 인한 추가 비용 가능성 명시 |

## 표현 사전

> 초안 작성 시 상황에 맞는 표현을 여기서 꺼내 조합한다.
> 그대로 복사가 아니라, 맥락에 맞게 주어/목적어를 바꿔 자연스럽게 쓴다.
> 부장님 메일이 추가될 때마다 새 표현을 누적한다.

### 감사·인사

| 상황 | 표현 |
|------|------|
| 서비스 이용 감사 | "그동안 저희 와트의 ~ 서비스를 이용해 주셔서 진심으로 감사드립니다." |
| 관심 감사 | "저희 와트에 관심 가져 주셔서 감사합니다." |
| 미팅 후 감사 | "바쁘신 중에도 시간 내어 주셔서 감사드립니다." |

### 관계 유지·재연결

| 상황 | 표현 |

## 템플릿 누적 규칙

사용자가 부장님 메일 원문을 제공하면:
1. 해당 메일의 **유형**을 판단 (기존 템플릿과 같으면 보강, 새로우면 추가)
2. **유형별 템플릿** 섹션에 새 템플릿 추가 (변수는 `{}`로 처리)
3. 새로운 표현이 있으면 **표현 사전**에 추가
4. 기존 원칙에 맞지 않는 새 패턴이면 **5대 원칙** 검토·보완
