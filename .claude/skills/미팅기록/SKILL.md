# 미팅기록

## 실행 순서

### Step 1: 기본 정보 수집
한 번에 다 알려줄 수 있으면 이 형식으로 받는다:
> 📥 **상황**: 언제 누구랑 만났는지
> 🔄 **내용**: 무슨 얘기 오갔는지 (간단히라도)
> 📤 **특이사항**: 평소와 다른 요청 있으면 (없으면 생략)

아니면 아래 항목을 하나씩 확인한다 (모르면 "-"로 비워둠):
- 고객사명
- 미팅 일시
- 참석자 (고객 측 / 우리 측)
- 미팅 목적 (신규 제안 / 현황 파악 / CS / 재계약 등)

### Step 2: 미팅 내용 정리
사용자가 말하는 미팅 내용을 받아 아래 구조로 정리한다:

```markdown
# [고객사명] 미팅 기록

**일시**: YYYY-MM-DD
**장소**:
**참석자**:
**목적**:

## 주요 논의 내용

## 고객 니즈 / 요청사항

## 우리 측 제안/대응

## 고객 반응/감성 메모
> 고객이 특히 반응하거나 감정이 변화한 순간 기록. 다음 제안·설득 포인트로 활용.
> 예: "레퍼런스 보고 표정 밝아짐" / "가격 언급 시 경직" / "음성인식 데모 보고 바로 질문 쏟아짐"
-

## 다음 액션
- [ ] (담당: 나 / 기한: )
```

### Step 3: 파일 저장

> 📌 **파일명 공통 규칙**: 날짜 뒤에 **(요일)** 병기 필수. 요일은 한 글자(월/화/수/목/금/토/일). 예: `2026-04-09(목)`

**① 분석 결과** → `고객관리/[고객사명]/YYYY-MM-DD(요일)_[주제]_[참여자].md`
- 폴더 없으면 자동 생성
- 파일명 규칙: `YYYY-MM-DD(요일)_[주제요약]_[참여자].md`
  - 주제: 핵심 키워드 2~3단어 (예: 직캠프로젝트방향, 일산방문계획)
  - 참여자: 우리 측 + 고객 측 (사용자가 알려준 경우) 이름 포함 (예: 김영준이사_김우곤부장, 박철수팀장_김영준이사)
  - 예시: `2026-04-01(수)_직캠프로젝트방향_김영준이사_김우곤부장.md`

**② 원본 파일** → `..private/미팅녹음/YYYY-MM-DD(요일)_HHMM_[주제]_[참여자].txt`
- 사용자가 녹음 파일(txt 등)을 제공한 경우, **원본을 그대로** `..private/미팅녹음/`에 복사한다
- 파일명 규칙: `YYYY-MM-DD(요일)_HHMM_[주제]_[참여자].txt` (HHMM = 24시간 형식, 예: 1517 = 오후 3시 17분)
- 시간은 원본 파일명이나 녹음 내용에서 파악. 모르면 사용자에게 확인
- 같은 미팅에 **여러 버전**이 있으면 뒤에 `_내기록` 등 구분자 추가 (예: `..._내기록.txt`)
- 여러 고객사가 하나의 녹음에 포함된 경우: 파일 1개로 유지, 주제에 고객사 병기 (예: `2026-04-01(수)_1517_내부회의_도공_한일_현대차_김영준이사_김우곤부장.txt`)

> 💡 요일 산출은 날짜로부터 계산하여 자동 병기. 한 자리 한글(월/화/수/목/금/토/일) 사용.

### Step 4: TODO.md 후속 업무 추가
다음 액션 항목이 있으면 TODO.md의 **내일/다음 할 일** 섹션에 자동 추가한다.

### Step 4-2: 신규 업종 고객 시 가설 확인
처음 만나는 업종의 고객이라면 미팅 기록 하단에 `## 가설 검증` 섹션을 추가한다:

```markdown
## 가설 검증 (신규 업종 시)
- 미팅 전 가설: [이 고객은 이런 니즈가 있을 것이다]
- 맞은 것:
- 틀린 것:
- 틈(내가 더 알아야 할 것):
```

참조: `스크립트/창업_사업개발/📋_분석요약.md` (박일용 가설→검증 프레임워크)

### Step 4-1: 고객 프로파일 업데이트
두 파일을 확인하고 변경사항을 반영한다:

**① `고객관리/[고객사명]/프로파일.md`** (우선)
- 담당자 이름·직책 파악됐을 때
- 관계 단계 변경됐을 때 (첫미팅 → 제안 등)
- 미팅 기록 테이블에 이번 미팅 한 줄 추가
- 파일 없으면 신규 생성

**② `참조자료/고객.md`**
- 신규 고객사 추가 시
- 도입 제품·계약 현황 변경 시

### Step 5: 저장 전 체크리스트
아래 항목이 모두 채워졌는지 확인하고, 빠진 항목은 사용자에게 물어본다:
- [ ] 고객사명 입력됨
- [ ] 미팅 일시 입력됨
- [ ] 참석자 (고객 측 또는 우리 측 중 하나 이상) 입력됨
- [ ] 주요 논의 내용 최소 1건 있음
- [ ] 다음 액션 최소 1건 있음 (담당자·기한 포함)
- [ ] 파일이 `고객관리/[고객사명]/YYYY-MM-DD_미팅.md` 경로에 저장됨
- [ ] TODO.md에 후속 업무 반영됨

### Step 6: 완료 보고
```
💾 저장: 고객관리/[고객사명]/YYYY-MM-DD_미팅.md
📋 TODO.md에 후속 업무 N건 추가
✅ 체크리스트 N/7 항목 완료
```
