# CS케이스 스킬

고객 CS 문의를 접수하여 **간결한 케이스 파일 + 영업부 공유문**을 생성한다.

## 작성 원칙

- CS 파일은 **핵심 사실만** (~50줄) — 한눈 요약 + 기본 정보 + 이슈 상세 + 공유문
- 후속 조치 → **TODO.md**에만 기록
- 교훈·패턴 → **decisions.md**에 축적
- 영업 문의 → **고객 프로파일**에 기록
- 통화 전문·가설·시나리오·위험 평가 등 사전 분석 → **파일에 저장하지 않음** (접수 시 화면 안내만)
- 추측·가정은 `❌ 미파악` 명시. 사실과 가설 분리
- 통화 녹취·메시지 원문이 있으면 이슈 상세의 **증상** 항목에 원문 인용 포함

## 참조 문서

- 제품 스펙·FAQ: `..private/참조자료/제품.md`
- 고객사 정보: `..private/참조자료/고객.md`
- CS 대응 패턴·노하우: `..private/참조자료/decisions.md`
- 두레이 과거 이력: `..private/두레이_아카이브/고객사별/`
- CS 폴더 규칙: `CS/CLAUDE.md`
- CS 인덱스: `CS/CS_인덱스.md`

## 상태 업데이트 흐름

### U-1: 대상 케이스 식별
CS 번호 또는 고객사명+증상으로 기존 케이스 파일을 찾는다.

### U-2: 케이스 파일 갱신
- `현재 상태` 변경 + 이슈 상세 업데이트
- `## 영업부 공유문` 현재 상태 기준 전면 재생성

### U-3: 연쇄 갱신
- `CS/CS_인덱스.md` — 해당 행 상태 변경 + 상단 현황 숫자 갱신
- `고객관리/[고객사]/프로파일.md` — CS 이력 테이블 상태 갱신
- `TODO.md` — 해당 후속 조치 항목 완료 처리

### U-4: 완료 보고
```
✅ CS 상태 변경: CS-[번호] [이전] → [새 상태]
📊 인덱스·프로파일·TODO 갱신
🗒️ 영업부 공유문 재생성
```

## 실행 순서 (신규 접수)

### Step 1: 정보 수집

> **고객사**: 어디에서 (본사/지사)
> **담당자**: 이름, 직급, 전화번호
> **증상**: 뭐가 안 되는지
> **경위**: 누가 언제 알려줬는지 (채팅 복붙, 메일, 통화 등)

### Step 2: CS 번호 부여

`CS/` 폴더 확인 → `CS-[3자리]` 순번 (예: CS-005)

### Step 2.5: 유사 이슈 탐지

접수 전 **3곳 검색** → CS 파일 `## 유사 이슈` 섹션에 **간결한 bullet 리스트**로 기록:
1. `CS/` 폴더 — 기존 CS 케이스
2. `..private/두레이_아카이브/고객사별/` — 과거 이력
3. `..private/참조자료/decisions.md` — 대응 패턴

결과 있으면 사용자에게 보여주고 CS 파일에 기록. 없으면 "유사 이슈 없음 (신규 유형)" 기록.

### Step 3: 유형 분류

| 유형 | 설명 |
|------|------|
| **기능오류** | 동작 안 함, 저장 안 됨, 크래시 |
| **설정연결** | 네트워크, 로그인, 서버 접속 |
| **사용법문의** | 기능 사용 방법 |
| **하드웨어** | 기기 파손, 배터리, 물리적 문제 |
| **성능** | 느림, 끊김, 화질 저하 |

### Step 4: 케이스 파일 작성

**파일명**: `CS/YYYY-MM-DD(요일)_[고객사]([지사])_[담당자]([직급])_[전화번호]_CS-[번호].md`
- 예: `CS/2026-04-03(금)_한국전력(서울본부)_정원식(대리)_010-9473-0720_CS-001.md`

**템플릿 (~40~50줄)**:

```markdown
# CS-[번호] | [고객사]([지사]) | [담당자] [직급] | [간략제목]

> **한눈 요약**
> - **고객**: [고객사(지사) 담당자 이름 직급 (전화번호)]
> - **증상**: [한 줄 증상]
> - **원인**: [✅ 파악 — 한 줄 / ❌ 미파악 — 1순위 의심]
> - **조치**: [현재 상태 → 다음 액션]
> - **비고**: [추가 정보 있으면 (없으면 생략)]

### Step 7: TODO.md에 후속 조치 추가

### Step 8: 반복 패턴 체크

- 같은 제품+유형 **2건 이상** → `decisions.md` CS 대응 패턴에 기록
- 같은 제품+유형 **3건 이상** → `CS/스마트글라스/CS_패턴_FAQ.md` + `CS/제품피드백/` 자동 생성

### Step 9: 영업부 공유문 자동 생성

CS 파일 하단 `## 영업부 공유문` 을 **항상 자동 생성** (신규·상태 업데이트 모두).

**제목**: ✅ 해결 → `[고객사(지사) CS 공유]` / 🟡/❌ → `[고객사(지사) CS 진행상황]`

**상태 헤더** (제목 아래 2줄, `원인:`와 `조치:` 각각 독립 줄):
```
원인: 🔴/🟢 [미파악/파악]
조치: ⬜/🟡/🟢 [다음 액션 한 문장만]
```

**본문 3단**:
```
1. 담당자: [이름] [직급] ([전화번호])
2. 접수 (MM-DD): [한 줄 증상]    ← 다이슈면 ① ... ② ... / 신규(MM-DD): ③ ...
3. 맥락: [이슈 + 해결방법/방향만 — 원인 추측·선례·배경 금지]
```

**규칙**:
- 콜론 라벨(`원인:` `조치:`)은 **각각 줄 바꿈**. 과거에는 `|`로 한 줄 처리했으나 가독성 위해 2줄 분리 (2026-04-20~)
- 조치는 **다음 액션 한 문장만** (과거 진행 제외)
- 가설은 공유문에 포함하지 않음
- 상태 변경 시 전면 재작성
- **맥락 작성 원칙 (극단적 간결)**: "이슈 + 해결방법/방향"만. 원인 추측, 선례, 배경, 경위, 통화 맥락 전부 금지. 헤더와 중복되는 상태 이모지도 금지 (2026-04-20 사용자 피드백: "이슈 설명은 딱 이슈랑 해결방법만, 나머지는 다 필요없어")
  - **단일이슈**: 한 줄 `[이슈] → [해결방법 or 다음 단계]`
    - 예 (잘됨): `3. 맥락: LTE 동글 연결 불가 → 동글 2개 교환으로 해결`
    - 예 (잘못): `3. 맥락: 고객사 동글이 타사 개통분으로 4/8 만료 → 4/10 마석역 현장에서 교환 → 정상 확인 (원인 배경 장황)`
  - **다이슈**: 이슈별 한 줄 bullet, `이슈N: [이슈] → [해결방법/방향]`
    - 예 (잘됨): `- 이슈1: TBM 항목 미표시 → 관리자 등급 일괄 전환으로 해결`
    - 예 (잘못): `- 이슈1: 계정이 '사용자 등급'으로 설정돼 TBM 권한 자체가 없었던 게 원인 → 관리자 등급 일괄 전환으로 해결 (원인 설명 불필요)`

### Step 10: 완료 보고

📊 인덱스 갱신: CS/CS_인덱스.md

🔁 반복 패턴: [감지됨 / 해당 없음]
```
