# 문서점검 (Garbage Collection)

> 하네스 엔지니어링 3기둥 중 "가비지 컬렉션" — 문서 간 불일치를 주기적으로 탐지·보고

## 목적
여러 파일에 분산된 고객·영업·TODO 정보가 서로 어긋나는 것을 잡아낸다.
**수정은 보고 후 사용자 승인 시에만** 진행한다.

## 검증 대상 파일
1. `TODO.md` — 할 일, 기한, DRI
2. `영업기록/파이프라인.md` — 고객사별 단계, 다음 액션, 기한
3. `..private/참조자료/고객.md` — 고객사 정보, 담당자, 진행 현황
4. `고객관리/[고객사명]/프로파일.md` — 개별 고객 프로파일 (있는 것만)
5. `고객관리/[고객사명]/*.md` — 최근 미팅 기록 (최신 1건만)

## 실행 순서

### Step 1: 전체 파일 읽기
위 5개 카테고리의 파일을 모두 읽는다. 고객관리/ 폴더는 Glob으로 존재하는 것만.

### Step 2: 교차 검증 — 5가지 체크

#### Check 1: 기한 정합성
- TODO.md의 기한 vs 파이프라인.md의 기한 → 같은 건인데 날짜가 다르면 플래그
- **오늘 날짜 기준** 이미 지난 기한인데 완료 표시 안 된 항목 → 플래그

#### Check 2: 고객사 단계 정합성
- 파이프라인.md의 단계(리드/제안/POC 등) vs 고객.md의 진행 상태 → 불일치 시 플래그
- 파이프라인.md에는 있는데 고객.md에는 없는 고객사 → 플래그

#### Check 3: 담당자·연락처 정합성
- 고객.md의 담당자 vs 고객관리/프로파일.md의 담당자 → 불일치 시 플래그
- 프로파일.md가 없는 활성 고객사 → "프로파일 생성 필요" 플래그

#### Check 4: 완료 항목 정리
- TODO.md에서 ✅ 완료된 항목이 7일 이상 지남 → "완료 섹션으로 이동 가능" 보고
- 파이프라인.md에서 계약완료된 건 → "성사 테이블로 이동 가능" 보고

#### Check 5: 최근 미팅 vs 파이프라인 갱신
- 고객관리/ 폴더에 최근 7일 내 미팅 기록이 있는데 파이프라인.md의 해당 고객 "다음 액션"이 미팅 이전 내용 → "파이프라인 업데이트 필요" 플래그

### Step 3: 결과 보고

**"성공은 조용히, 실패만 시끄럽게"** 원칙 적용:
- 불일치 0건 → 한 줄로 끝: `✅ 정합성 체크 완료 — 불일치 없음`
- 불일치 있으면 → 아래 형식으로 보고:

```
## 🔍 정합성 체크 결과 (YYYY-MM-DD)

### ⚠️ 불일치 발견: N건

| # | 유형 | 파일 A | 파일 B | 내용 | 제안 |
|---|------|--------|--------|------|------|
| 1 | 기한 불일치 | TODO.md | 파이프라인.md | 세스코 기한: TODO 04-09 vs 파이프라인 05-02 | 파이프라인 기준으로 통일 |

### 📋 정리 가능 항목: N건
- [ ] TODO.md 완료 항목 3건 → 완료 섹션 이동
- [ ] 파이프라인 계약완료 1건 → 성사 테이블 이동
```

### Step 4: 수정 실행 (승인 시)
사용자가 "수정해줘" 또는 특정 항목을 승인하면 해당 파일을 업데이트한다.
**사용자 승인 없이 절대 자동 수정하지 않는다.**

## 특정 고객사 지정 시
인자로 고객사명이 들어오면 해당 고객사 관련 파일만 집중 체크한다.
