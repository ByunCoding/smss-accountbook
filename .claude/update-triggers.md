# 자동 업데이트 트리거 상세

> CLAUDE.md에서 분리된 상세 규칙. 필요할 때만 참조.

## 트리거 → 업데이트 대상

| 이런 일이 생기면 | 이걸 업데이트해라 |
|----------------|-----------------|
| 신규 고객사 확정 | `..private/참조자료/고객.md` + `고객관리/[고객사명]/프로파일.md` 생성 |
| 고객 미팅 완료 | `고객관리/[고객사명]/YYYY-MM-DD_미팅.md` 생성 + `프로파일.md` 담당자·상태 업데이트 |
| 딜 단계 변경 | `영업기록/파이프라인.md` 해당 고객사 행 업데이트 |
| CS 케이스 해결 | `CS/` 에 케이스 파일 저장 + 반복 패턴이면 `..private/참조자료/decisions.md`에 추가 |
| 영업 제안서 작성 | `영업기록/` 에 저장 + `파이프라인.md` 단계 업데이트 |
| 공공입찰 결과 확인 | `영업기록/파이프라인.md` 결과 반영 (낙찰/탈락) |
| 제품 스펙 변경·확인 | `..private/참조자료/제품.md` 업데이트 |
| 경쟁사 새 정보 입수 | `..private/참조자료/경쟁사.md` 업데이트 |
| 업무 패턴·노하우 발견 | `..private/참조자료/decisions.md` 에 `### YYYY-MM-DD: 제목` 형식으로 추가 |
| 월말 성과 정리 | `성과기록/YYYY-MM_성과기록.md` 생성 또는 업데이트 |
| 새 유튜브 스크립트 추가 | `스크립트/CLAUDE.md` 규칙 따라 처리 |
| 새 도메인 용어·약어 등장 | `..private/참조자료/용어.md` 해당 섹션에 추가 |

## "CLAUDE.md 업데이트해줘" 요청 시 수행 순서
1. 변경된 내용 파악 (새 파일, 새 고객, 새 패턴 등)
2. 위 테이블 기준으로 해당 파일들 업데이트
3. CLAUDE.md 자체의 참조 경로도 갱신
4. 실제와 다른 규칙이 발견되면 이 테이블도 수정
