# 개선점 분석

현재 앱의 개선 가능한 부분을 분석합니다.

1. index.html을 분석하여 다음을 확인합니다:
   - UX 개선점 (사용성)
   - 시각적 개선점 (디자인 일관성)
   - 성능 개선점 (불필요한 리렌더링, DOM 접근)
   - 코드 품질 (중복, 미사용 코드)

2. 각 항목에 대해 난이도(쉬움/보통/어려움)와 효과(낮음/중간/높음) 표시

3. 우선순위별로 정리하여 표로 보고

이미 구현된 기능:
- PWA, 다크모드, 로딩 스피너, 월 탭 UI
- 입력 유효성 검사, 삭제 확인
- 필터 초기화, debounce, 빈 상태 UI
- 카테고리 자동 추천 (키워드 기반)
- 토스트 safe-area
